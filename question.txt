Q: Why not store token in localStorage or cookies?

👉 We generally avoid localStorage because it is vulnerable to XSS attacks.
If an attacker injects a script, they can easily read the token and steal it.

👉 Cookies are also not fully safe because of CSRF attacks, where a malicious site can send requests automatically using the stored cookie.


Simple Answer

Q: What about storing token in memory?

👉 Storing token in memory (like variables/state) is more secure than localStorage and cookies because:

• It is not accessible globally
• It is cleared when page refreshes

Advanced Backend Topic

Refresh Token Rotation (What, Why, When)

What:
Refresh token rotation is a technique where a new refresh token is generated every time it is used, and the old refresh token is invalidated.

Why:
It is used to improve security by preventing token reuse (replay attacks).
If a refresh token is stolen, it cannot be used multiple times.
It also helps in detecting suspicious activity.

When:
It is used when the access token expires.
The client sends the refresh token to the server, and the server issues:
• a new access token
• a new refresh token

while invalidating the old refresh token.


One-line answer:

👉 “Refresh token rotation enhances security by replacing old refresh tokens with new ones on every use, preventing replay attacks.”
